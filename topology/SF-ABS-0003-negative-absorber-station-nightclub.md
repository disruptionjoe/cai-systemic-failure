# SF-ABS-0003 - Station Nightclub Negative Absorber Test

## Boundary

This negative absorber uses the NIST Station nightclub fire source set to test
whether `SF-SCHEMA-CANDIDATE-0001` refuses a real public-source systemic
failure when the stronger public-source explanation is fire/life-safety code,
material, sprinkler, pyrotechnic, and egress protection rather than
source-backed assumption reuse under changed conditions.

It does not create an accepted topology schema, fire-safety model, building-code
conclusion, legal conclusion, enforcement position, public recommendation, CAI
doctrine, intervention plan, or owner routing instruction.

No confidential material, human subjects, field work, account access, external
contact, posting, spending, publication outside normal GitHub versioning, or
deployment was used.

## Absorber Question

The provisional candidate should refuse this case as positive evidence if its
gates are doing real work.

The NIST source set centers pyrotechnics igniting polyurethane foam, rapid fire
spread, no installed sprinklers, main-entrance crowding, evacuation limits,
model-code application, fire-protection requirements, material restrictions,
and inspection or emergency-response recommendations. That is a
fire/life-safety code and protection-system absorber for this repository, not a
review-chain assumption-reuse record.

The source set does create a useful warning pressure: a building/fire-safety
case can involve code history, existing-building treatment, inspection, and
jurisdictional responsibility without qualifying for the candidate. Positive
fit still requires a source-backed old assumption reused under changed
conditions, a visible validation burden, delegation or review fragmentation,
and an owner-visible correction route.

## Verdict

`SF-SCHEMA-CANDIDATE-0001` refuses this case as positive evidence. That refusal
is useful after `SF-0005` because it shows the candidate can accept one
source-backed building/fire-safety review-chain record without automatically
claiming every public building or fire-safety failure.

The candidate remains provisional and local to review-chain failures involving
source-backed assumption reuse under changed conditions, visible validation
burden, affected standing, absorber control, and correction-route discipline.
This negative absorber does not justify schema acceptance.

## Candidate Revision Pressure

Keep the candidate field set as-is, but add one refusal note to the candidate:
when official sources center domain-code application, fire/life-safety
protection, material flammability, suppression, pyrotechnic, evacuation, or
similar domain-specific remedy paths rather than source-backed assumption reuse
under changed conditions, the case should be recorded as an absorber or
separate topology question, not positive evidence for this candidate.

This note must remain source-gated. It does not authorize this repository to
make building-code, fire-safety, emergency-response, legal, regulatory,
enforcement, public-safety, or domain-remedy conclusions.
