# SF-SQA-0001 - Source-Quality Falsifier

## Boundary

This falsifier tests whether `SF-SCHEMA-CANDIDATE-0001` refuses cases that
look like review-chain failures only because weak source material, retrospective
storytelling, or hindsight reconstruction makes the candidate fields appear to
fit.

It uses the repository's existing records, source pointers, governance gate,
candidate gate, and acceptance-bar review. It does not add a new real-world
failure record, accept a topology schema, create a universal systemic-failure
ontology, prescribe a domain remedy, issue a public recommendation, authorize
human or field research, or move another owner's work.

No confidential material, human subjects, field work, account access, external
contact, posting, deployment, spending, publication outside normal GitHub
versioning, or external action was used.

## Falsifier Class

The acceptance-bar review names source-quality or hindsight artifact as an
untested falsifier class. This class matters because the candidate can only be
useful if it refuses attractive pattern matches when the source packet cannot
support the fields without the repository inventing links after the fact.

A source-thin case may still describe a real failure. It may even contain
affected people, organizational complexity, warnings, or later expert
narratives. That is not enough for positive fit. The candidate requires public,
licensed, safely shareable, or synthetic material that can carry the specific
source context, changed condition, validation burden, delegation or review
fragmentation, absorber, and correction-route fields.

## Refusal Fixture

The refusal fixture is intentionally minimal and repository-local:

- a public account says an organization ignored warnings before a failure;
- later commentary describes an old assumption and a missed validation burden;
- affected people and institutional owners are visible at a generic level;
- no primary, official, licensed, or otherwise durable source packet names the
  assumption's source context, the changed condition, the review owner, the
  validation burden, or the owner-visible correction route; and
- the repository would have to reconstruct those links from hindsight to make
  the candidate fields pass.

This fixture is not a claim about any real case. It is a source-quality test of
the candidate's refusal behavior.

## Verdict

`SF-SCHEMA-CANDIDATE-0001` refuses the fixture as positive evidence.

The refusal preserves source-quality discipline: the candidate may screen only
records that can carry their fields from public, licensed, safely shareable, or
synthetic material. If source material is weak, media-only, retrospective,
source-thin, or dependent on hindsight reconstruction, the case belongs as a
lead, inquiry, absorber, or separate topology question. It does not validate
the review-chain candidate.

This falsifier strengthens the candidate's boundary but does not justify schema
acceptance.

## Candidate Revision Pressure

Keep the candidate field list unchanged. Add a refusal note to the candidate:
source-thin or hindsight-built pattern matches must be refused as positive
evidence even when they resemble the review-chain shape.

The note should remain a source-quality control, not a new research program.
It does not authorize external contact, field research, account access,
scraping, publication, public claims, domain recommendations, or another
owner's work.
